NADA 4 - Repositories

Features we want to have

A repository represents a collection of studies in a catalog. A catalog can have one or more repositories. There are two types of repositories:

a. Publish/Unpublish: an unpublished repository is not visible on the frontend of the website

b. Repository Language****: We might have catalogs with different languages. e.g. one english catalog and one Arabic catalog. With the repository language attribute, we can change the interface language to match the repository language.

c. Repository Identification: This should always be a alphanumeric string with no spaces, no accentuated characters. NADA creates a folder for the repository using this id and it should be a string that can be a folder name.

d. Repository Type {Internal, External}:

----------------------

Repository Users and Roles [NOTE: This is not valid anymore]

Before a user can manage a repository, user must be assign to a repository using one of the two roles available:

  • Catalog Administrator: The person has full admin rights to the repository and can manage all areas of the repository
  • Catalog Licensed Survey Requests Administrator: This role limits the person to edit only the licensed survey requests for the selected repository only.


Repository Users and Roles

User roles are set at a global level. At the repository level you can only select what areas a person can access. The areas/sections available are:

a. Catalog

b. Reports

c. Licensed Requests
