NADA 4 - User Management

Global User Roles

User [access to site frontend only]

Admin [super admin, access to all areas of the website]

Catalog-admin [can manage all studies from all repositories] *

Lic-req-admin [can manage all licensed requests from all studies from all repositories] *

Report-admin [can manage all reports for all studies from all repositories] *

Study-admin_[sid] [can manage a single study only]

Limit user access to a repository

To limit user roles to a repository, add user to the repository and their global roles will be specific to that repository only.

For example, Catalog-admin role has admin rights on all studies from all catalogs. If user is assigned to the repository LSMS then he can only access LSMS studies.

Configuration setting to disable default access to all catalogs: For the multi-catalog system, a global catalog-admin or other roles will not have full access to all studies. User will have access to the repository that he is assigned to and will have the roles assigned at the repository level.

For example, if the user X with role CATALOG-ADMIN has been assigned to LSMS with permissions to only manage reports. he will not be able to access other sections that were granted at the global level.

The sections that are accessible at the repository level are:
[x] Catalog maintenace
[x] Licensed request administration
[x] Reports

This allows to make it possible for a user to have different roles for different repositories.

Note: If a user has LIC-REQ-ADMIN rights, he will not be able to access a catalog maintenance page even he is given the rights at repository level.
